# 实现多智能体协同——手动

**人为操控每一个智能体的上下文窗口。**



**ClaudeLink**具备多个终端以及多个开发商智能体协作，利用IU窗口实现：（1）交流内容可读性；（2）实时监控各智能体的状态；（3）统一/逐个发送指令。

其余内容可参考 ClaudeLink 的 README.md。

官方GitHub网址：[RBJGlobal/claudelink: The hub where your AI agents connect. Multi-terminal Claude Code agent communication via MCP.](https://github.com/RBJGlobal/claudelink/tree/main)



下面是利用Claude Code自带的MCP实现多终端。

## 🪟 Windows 完整操作指南

### 步骤 1：启动 MCP 内存服务器（独立终端）

**打开第一个命令提示符（CMD）或 PowerShell**，保持运行：

```
cd D:\myproject
npx @modelcontextprotocol/server-memory --memory-path ./shared_memory.json
```

> ⚠️ **保持这个终端一直运行**，不要关闭！它是所有 Agent 的"共享大脑"。

------

### 步骤 2：终端 1 - 开发者 Agent

**新开一个命令提示符窗口**：

```cmd
cd D:\myproject

# Windows CMD 设置环境变量
set MCP_CONFIG={"memory":{"command":"npx","args":["@modelcontextprotocol/server-memory","--memory-path","./shared_memory.json"]}}

# 启动 Claude Code
claude
```

**或者用 PowerShell**（推荐，语法更清晰）：

```powershell
cd D:\myproject
$env:MCP_CONFIG='{"memory":{"command":"npx","args":["@modelcontextprotocol/server-memory","--memory-path","./shared_memory.json"]}}'
claude
```

------

### 步骤 3：终端 2 - 审查者 Agent

**再开一个新命令提示符窗口**：

```cmd
cd D:\myproject
set MCP_CONFIG={"memory":{"command":"npx","args":["@modelcontextprotocol/server-memory","--memory-path","./shared_memory.json"]}}
claude
```

------

### 步骤 4：终端 3 - 测试者 Agent（可选）

**再开一个新命令提示符窗口**：

```cmd
cd D:\myproject
set MCP_CONFIG={"memory":{"command":"npx","args":["@modelcontextprotocol/server-memory","--memory-path","./shared_memory.json"]}}
claude
```

------

## 🤖 在 Claude Code 中协作

### 终端 1（开发者）输入：

```
请把当前项目拆解成 3 个开发任务，写入共享内存
```

### 终端 2（审查者）输入：

```
读取共享内存中的任务列表，审查第一部分任务，提出改进建议
```

### 终端 3（测试者）输入：

```
读取共享内存中的任务列表，为第二部分任务设计测试用例
```

## 💪在实践中总结的经验

### 核心操作：

claude code 安装：

```cmd
claudelink init --claude --global
```

然后进入到项目文件夹里：

```cmd
cd your-project

# 选择 npx claudelink init
npx claudelink init                       # Claude Code (.mcp.json + CLAUDE.md)
npx claudelink init --codex               # Codex CLI (AGENTS.md + TOML snippet)
npx claudelink init --gemini              # Gemini CLI (.gemini/settings.json + GEMINI.md)
npx claudelink init --goose               # Goose (.goosehints + config snippet)
npx claudelink init --codex --gemini      # stack flags additively
npx claudelink init --all                 # all four clients in one project
```

发送指令，要求扮演角色：

**Terminal 1** (Claude Code, advisor)

> "Register as a code reviewer working on the auth module."

**Terminal 2** (Codex CLI, implementer)

> "Register me as a developer. Send a message to the reviewer asking for the last hot-spot list."

**Terminal 3** (Gemini CLI, validator) — or just leave the Command Center open.

> "Register me as a tester. Watch for completed work and run validation passes."

当然也可以是中文。具体指令建议先让AI给予，熟练了再手动创建。

不到一分钟，评审人员的回复就出现在二号终端中，开发人员据此进行处理，公告板同步展示变更内容。完成注册后，你未在二号、三号终端输入任何指令。命令中心页面可实时显示全部三台对等节点，无论各节点另一端搭载何种模型。[http://127.0.0.1:7878](http://127.0.0.1:7878/)

### ClaudeLink的优点：本地优先安全机制

ClaudeLink 的设计理念是无缝融入你的开发环境，而非成为一款需要你额外信任的SaaS管理面板：

无云端、无服务器。所有状态数据全部存储在本地设备路径 ~/.claudelink/nexus.db 中。
无遥测采集、无数据分析、无后台回传程序。你可直接查阅源代码——不存在需要手动关闭的数据上报功能，因为程序本身不会向外传输任何数据。
无需账号、无需注册、无需API密钥。ClaudeLink不会与任何外部服务通信。各类智能代理仍按照原有逻辑对接各自的大模型服务商；ClaudeLink仅作为代理之间的连接中间层。
默认仅本地回环访问。命令中心绑定地址为127.0.0.1:7878。除非你手动开启对应权限，否则外部设备无法访问（该跨设备功能计划在v1.2版本上线，局域网绑定将配套Bearer令牌鉴权机制）。
完全开源，采用MIT开源协议。全部代码均托管于本代码仓库。代码可审计、可分支复刻，企业用户也可自行编译定制版本以满足合规要求。
沙箱化终端交互。自动调度工具通过iTerm2、tmux开放的AppleScript、send-keys接口向终端输入指令，逻辑与Bash脚本完全一致。无需辅助无障碍插件、无需内核钩子，也不会弹出权限申请弹窗。

### ClaudeLink拥有Web UI

这个Web UI有非常明显的缺陷。为了轻量化且能够在本地使用，官方使用了 **轮询 (Polling)** 作为通信方式。下面比较了四大主流的通信方式：

| 技术                         | 原理             | 实时性     | 服务器压力 | 典型应用       |
| :--------------------------- | :--------------- | :--------- | :--------- | :------------- |
| **轮询 (Polling)**           | 每隔几秒发请求   | 取决于间隔 | 高         | 简单通知       |
| **长轮询 (Long Polling)**    | 等待有数据才返回 | 近实时     | 中         | 聊天室         |
| **WebSocket**                | 双向持久连接     | 实时       | 低         | 在线游戏、聊天 |
| **SSE (Server-Sent Events)** | 服务器单向推送   | 实时       | 低         | 股票行情、通知 |

启动UI之后进入浏览器页面会每隔两秒刷新一遍，这个刷新会导致每隔两秒就会有一个命令行窗口弹出又闪退，这会直接阻断中文输入法以及强行阻断打开的某个页面。经过本人反复实验+查询官方文档+求助AI，得到了一个非常简单的方法，那就是**禁用当前这个页面的JavaScript**。

```
1. 地址栏输入：edge://settings/content/javascript
2. 点击 "不允许任何网站运行 JavaScript"
3. 输入你想要禁用的网址
```

这样刷新就暂停了。不过这只是缓兵之计，真正要查看的话还是要开启的，否则页面不会保留状态，即使你注册了Agent。好处就是在你写命令的时候关闭会非常爽，另外要保留浏览器中处理JavaScript的设置页面，这样操作更方便。另外记得刷新。

#### 实践结果表明

合作并不会陷入死循环，两者达到了共识（Consensus）。方案制定、代码书写等是一个开放性的主题[^1]，能够在这类主题上达成共识，从数学角度看，双方若不采用自然语言传递的方式转而采用向量传递的话，内部必然收敛。与开放性主题不同的是，纳什均衡（Oroojlooy and Hajinezhad ，2023[^2]），笼统的理解即在有限制的场景下达成收敛，少数服从多数，如投票。
