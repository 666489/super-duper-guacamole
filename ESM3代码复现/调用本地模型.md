## `ESM3.from_pretrained("esm3_sm_open_v1")` 的背后做了什么

**第一步：名称 → 构造函数的映射**

`"esm3_sm_open_v1"` 是一个字符串常量，它的值是 `"esm3-sm-open-v1"`，定义在 [pretrained.py:121](vscode-webview://1ivti5q4o0no53ll02igigsv3i42li27tuurau03sup9uqd46kvd/index.html?id=d15e31c3-23db-4892-b2b7-bb8b81c3884b&parentId=1&origin=31e8dbed-ac86-4b99-a01d-25f9ea4848b5&swVersion=4&extensionId=Anthropic.claude-code&platform=electron&vscode-resource-base-authority=vscode-resource.vscode-cdn.net&parentOrigin=vscode-file%3A%2F%2Fvscode-app&session=f1b64b52-0697-4e3d-b136-bc3cfe683cda)：

```python
LOCAL_MODEL_REGISTRY = {
    "esm3-sm-open-v1": ESM3_sm_open_v0,   # ← 这个
    ...
}
```

**第二步：构造函数创建模型架构并加载权重**

`ESM3_sm_open_v0` 做了两件事（[pretrained.py:101-116](vscode-webview://1ivti5q4o0no53ll02igigsv3i42li27tuurau03sup9uqd46kvd/index.html?id=d15e31c3-23db-4892-b2b7-bb8b81c3884b&parentId=1&origin=31e8dbed-ac86-4b99-a01d-25f9ea4848b5&swVersion=4&extensionId=Anthropic.claude-code&platform=electron&vscode-resource-base-authority=vscode-resource.vscode-cdn.net&parentOrigin=vscode-file%3A%2F%2Fvscode-app&session=f1b64b52-0697-4e3d-b136-bc3cfe683cda)）：

1. **搭建架构**：`ESM3(d_model=1536, n_heads=24, v_heads=256, n_layers=48, ...)`
2. **加载权重**：从本地文件读取 `.pth`

```python
state_dict = torch.load(
    data_root("esm3") / "data/weights/esm3_sm_open_v1.pth",
    map_location=device,
)
```

**第三步：权重文件的本地路径**

`data_root("esm3")` 返回的是 HuggingFace Hub 缓存目录：

```
D:\huggingface\hub\models--EvolutionaryScale--esm3-sm-open-v1\snapshots\47f0545b...
```

完整路径就是：

```
D:\huggingface\hub\models--EvolutionaryScale--esm3-sm-open-v1\snapshots\47f0545b...\
    data/weights/esm3_sm_open_v1.pth
```

### 为什么首次就能直接加载？

| 步骤                         | 发生了什么                                                   |
| ---------------------------- | ------------------------------------------------------------ |
| **首次** `from_pretrained()` | 看到 HuggingFace 模型名 `EvolutionaryScale/esm3-sm-open-v1` → 自动从 HF Hub **下载**到 `D:\huggingface\hub\` 缓存目录 → 加载 |
| **之后**                     | 检查缓存已存在 → 直接加载，不再下载                          |



### 调用镜像的原理

```
ESMC.from_pretrained("esmc_600m")      # 你写的代码
  │
  └─→ load_local_model("esmc_600m")   # esm/pretrained.py
        │
        └─→ ESMC_600M_202412(device)  # 构造函数
              │
              └─→ torch.load(
                    data_root("esmc-600") / "data/weights/..."
                  )
                    │
                    └─→ data_root("esmc-600")
                          │
                          └─→ snapshot_download(
                                repo_id="EvolutionaryScale/esmc-600m-2024-12"
                              )
                                │
                                └─→ huggingface_hub 自动读取
                                    os.environ["HF_ENDPOINT"]
                                      │
                                      ▼
                              https://hf-mirror.com  ← 你的镜像
```

`huggingface_hub` 库底层在发起 HTTP 请求时会自动检查 `HF_ENDPOINT` 环境变量，把 `https://huggingface.co` 替换为 `https://hf-mirror.com`。整个过程对你透明——代码里写的是 `from_pretrained("esmc_600m")`，路径全是自动解析的，跟用不用镜像无关。

Cell 4 的 SAE 权重除外，那是通过 `modelscope` 单独下载的（`snapshot_download` 走 `www.modelscope.cn`），不走 HF 镜像。

### 1. `repo_id` → 文件夹名的映射规则

`huggingface_hub` 内部会把 `repo_id`（如 `EvolutionaryScale/esm3-sm-open-v1`）标准化为缓存路径：

```
REPO_ID:   EvolutionaryScale/esm3-sm-open-v1
               ↓
CACHE_PATH: D:\huggingface\hub\models--EvolutionaryScale--esm3-sm-open-v1
                              └── type ──┘  └── namespace ──┘  └── repo ──┘
```

规则很简单：

| 原始字符 | 替换为                                      |
| -------- | ------------------------------------------- |
| `/`      | `--`                                        |
| 前缀     | `models--`（模型）或 `datasets--`（数据集） |

所以 `EvolutionaryScale/esmc-600m-2024-12` 就变成 `models--EvolutionaryScale--esmc-600m-2024-12`。

同一条规则也解释了你的 ESM3：

```
EvolutionaryScale/esm3-sm-open-v1
  → models--EvolutionaryScale--esm3-sm-open-v1
```

我已经在你的 `D:\huggingface\hub\` 里看到了这个文件夹，证实了这一点。

这个转换是 `huggingface_hub` 底层自动完成的，对 `snapshot_download` 调用者完全透明——你传 `repo_id`，底层自己算路径。

### 2. ModelScope 下载的模型怎么用？

这是关键点：**两个下载系统、两套缓存，互不通用**。

```
HF 镜像下载                         ModelScope 下载
     │                                    │
     ▼                                    ▼
huggingface_hub.snapshot_download()    modelscope.snapshot_download()
     │                                    │
     ▼                                    ▼
D:\huggingface\hub\                  C:\Users\Administrator\.cache\modelscope\hub\
  models--EvolutionaryScale--              biohub\ESMC-600M-sae-layer27-k32-codebook131072\
    esmc-600m-2024-12\                       layer_27.safetensors ← 手动读取
      data/weights/*.pth ← 自动加载
```

- **HF 下载的模型**（ESM3、ESMC-300M、ESMC-600M）：`esm` 库内置的 `from_pretrained()` 内部调用 `huggingface_hub`，自动在 `D:\huggingface\hub\` 中查找和加载。`HF_ENDPOINT` 只是改了下载时的 URL，缓存路径不受影响。

- **ModelScope 下载的模型**（SAE 权重）：`esm` 库**不认识** ModelScope 的缓存路径，不能走 `from_pretrained()`。所以 Cell 4 用的是 `safetensors` 直接手动读取。

  ```python
  sae_dir = "C:/Users/.../.cache/modelscope/hub/models/biohub/ESMC-600M-sae-layer27-k32-codebook131072"
  with safe_open(f"{sae_dir}/layer_27.safetensors", framework="pt") as f:
      ...
  ```

### 总结

|                   | HF 镜像                          | ModelScope                        |
| ----------------- | -------------------------------- | --------------------------------- |
| 缓存路径          | `D:\huggingface\hub\models--...` | `~\.cache\modelscope\hub\...`     |
| 加载方式          | `from_pretrained()` 自动         | 手动 `safetensors` / `torch.load` |
| Cell 1/2/3/4 用到 | ESM3、ESMC-300M、ESMC-600M 基座  | Cell 4 的 SAE 权重                |

这是你当前 notebook 不需要改动的现状。ModelScope 的模型就是得手动读——除非你把它复制或软链接到 HF 的缓存目录中，但这只是权宜之计，手动加载更清晰。
