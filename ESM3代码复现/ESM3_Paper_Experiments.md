## Cell 8: 预测结构 vs 实验结构 RMSD 对比

### 1. 实验目的

从 PDB 晶体数据库中选取已知结构的蛋白质，使用 ESM3 从序列**从头预测**（ab initio，不依赖任何同源模板，仅从氨基酸序列出发）三维结构。将预测结构与实验结构进行**刚性叠合**，计算 RMSD，评估 ESM3 1.5B 的结构预测精度。

### 2. 核心概念 —— 逐词解释

#### 2.1 刚性叠合 (Rigid-Body Superposition)

**刚性叠合**是一种将两个三维结构在空间中**对齐**的数学方法。

- **刚性**：结构本身不变形——不拉伸、不弯曲、不扭曲
- **平动**：整体移动位置（把两块积木搬到一起）
- **旋转**：整体改变朝向（把积木转到同一个角度）

目的是消除两个结构之间无意义的整体差异（比如坐标系原点在左边还是右边），只比较它们真正的**形状差异**。

```
预测结构 (任意朝向)         实验结构 (另一个朝向)
      ╲                         ╱
       ╲    旋转 + 平动         ╱
        ╲   ←──────────→      ╱
         ╲                    ╱
          ───── 叠合后 ─────
           剩下的差异 = 真正的结构误差
```

数学上，刚性叠合寻找一个**旋转矩阵 R** 和一个**平移向量 t**，最小化叠合后两个结构之间的坐标差平方和。叠合后剩下的就是结构预测误差，用 **RMSD** 表示。

#### 2.2 RMSD (Root Mean Square Deviation，均方根偏差)

RMSD 是衡量两个结构之间差异的最常用指标，单位是 **Å**（埃，1Å = 10⁻¹⁰ 米，约等于一个氢原子的直径）。

计算公式：

```
RMSD = sqrt( (1/N) × Σ ||pred_i - exp_i||² )

其中：
  N          = 蛋白序列长度（残基数）
  pred_i     = 预测结构中第 i 个 Cα 原子的坐标 (x, y, z)
  exp_i      = 实验结构中第 i 个 Cα 原子的坐标
  ||...||    = 欧几里得距离（三维空间中两点间的直线距离）
  sqrt(平方均值) = 使得结果的单位与原始坐标一致（都是 Å）
```

**为什么只用 Cα 原子？** 每个氨基酸有多个原子（主链骨架 + 侧链），Cα 是连接主链和侧链的"枢纽"原子，每个残基恰好一个。只用 Cα 能捕捉主链折叠形态，同时避免侧链取向的随机性干扰。

**RMSD 的经验判据**：

| RMSD | 含义 |
|------|------|
| < 2Å  | 优秀：预测结构与实验结构高度一致 |
| 2~5Å  | 中等：大体折叠正确，局部有偏差 |
| 5~10Å | 较差：折叠拓扑可能出错 |
| > 10Å | 很差：基本未能正确预测折叠 |

#### 2.3 Cα 原子 (Alpha Carbon)

蛋白质主链由 N-Cα-C 三个原子重复连接构成骨架：

```
      R (侧链)
      |
...— N — Cα — C — N — Cα — C — ...
     |         |
     H         O
```

Cα 是每个氨基酸的"分叉点"——它连接主链和侧链，每个残基恰好一个。RMSD 计算通常只取 Cα 原子，因为：
- 每个残基一个 Cα，一一对应，可以直接比较
- 侧链原子数量因氨基酸而异（Gly 无侧链、Trp 有 10 个），不可直接对齐
- Cα 轨迹足以反映蛋白质的三维折叠

#### 2.4 Kabsch 算法

Kabsch 算法是求解刚性叠合最优旋转矩阵的具体方法（1976 年由 Wolfgang Kabsch 提出）。

**流程**：

```
1. 中心化:  Pc = P - mean(P)       ← 把两个结构都搬到原点
            Qc = Q - mean(Q)

2. 协方差:  H = Pc^T × Qc          ← 计算两个结构之间的相关性矩阵 (3×3)

3. SVD分解: H = U × Σ × V^T        ← 将协方差矩阵分解为旋转部分

4. 最优旋转: R = V × U^T           ← 这就是最优旋转矩阵
            (需检查 det(R) = +1，若为 -1 则修正镜像)

5. 叠合:    P_aligned = Pc × R + mean(Q)  ← 旋转后再搬回 Q 的位置

6. RMSD:    sqrt(mean(||P_aligned - Q||²))
```

**关键**：SVD 分解保证了 R 是在所有可能的旋转中使 RMSD 最小化的那个。

#### 2.5 ESM3 结构预测管线

ESM3 不是直接在实数坐标空间预测结构，而是通过 **VQ-VAE 离散码本**：

```
氨基酸序列
    │
    ▼
ESM3 Transformer (48 层, 1536 维)
    │
    ├─→ sequence_logits        (氨基酸, 33 维)
    ├─→ structure_logits       (结构 token, 4097 维)
    │        │
    │        ▼ argmax ← 每个位置选最可能的 token
    │   VQ-VAE 结构 token      (每个残基 = 1 个整数, 0~4095)
    │        │
    │        ▼ StructureTokenDecoder.decode()
    │   骨架坐标 (N, CA, C)    ← bb_pred: (B, L, 3, 3)
    │        │
    │        ▼ ProteinChain.from_backbone_atom_coordinates()
    │   atom37 坐标             ← infer_oxygen() 推断氧原子
    │
    └─→ pLDDT / pTM            (置信度)
```

**为什么用 VQ-VAE 而非直接预测坐标？**
- VQ-VAE（Vector Quantized Variational AutoEncoder）将连续的 3D 空间量化为 4096 个离散码本
- 离散化使得结构预测变成一个**分类问题**（选 token）而非回归问题（直接输出坐标）
- 优点：训练更稳定、支持条件生成（给定序列 → 采样结构）
- 代价：**精度上限受码本大小限制**——4096 个 token 约等于每残基 ~2.5Å 的理论精度上限

#### 2.6 pLDDT 与 pTM —— 置信度指标

这两个指标源自 AlphaFold2，被 ESM3 借用但**不可同等信赖**。

| 指标 | 全称 | 含义 | 范围 |
|------|------|------|------|
| **pLDDT** | predicted Local Distance Difference Test | 每残基的**局部**结构置信度 | 0~1，越高越好 |
| **pTM** | predicted Template Modeling score | **全局**结构置信度（整体折叠是否正确） | 0~1，>0.5=大致正确，>0.8=高置信 |

**LDDT 是什么？** Local Distance Difference Test 是一种不依赖叠合的局部结构评估指标——对每个残基检查其周围一定范围内的原子间距离是否与参考结构一致。pLDDT 是模型"猜测"自己的 LDDT 会有多高。

**TM-score 是什么？** Template Modeling score 衡量两个蛋白结构的全局相似度，对长度不敏感（不同于 RMSD）。一个随机蛋白对的 TM-score ≈ 0.17，完全相同 = 1.0。

**关键警告**：在 AlphaFold2 中（有 MSA 输入），pLDDT/pTM 与真实精度有强相关性。但在 ESM3 单序列模式下，本实验发现**高 pTM 不等于低 RMSD**（见结果部分）。

#### 3.2 PDB 文件解析逻辑

解析一个 PDB 文件需要提取三类信息：

**(a) 分辨率 (Resolution)**

```python
# 仅从 REMARK 2 行提取
for line in text.split("
")[:100]:
    if line.startswith("REMARK   2 RESOLUTION."):
        resol = float(line.split()[-2])  # 如 "2.07 ANGSTROMS."
```

分辨率是衡量晶体结构质量的指标，单位 Å。低分辨率 = 高质量（1.0Å 是原子级精度，3.0Å 只能看清主链走向）。

**(b) 氨基酸序列 (从 Cα 提取)**

```python
AA3TO1 = {"ALA":"A", "CYS":"C", "ASP":"D", ...}  # 三字母→单字母

for line in text.split("
"):
    if line.startswith("ATOM") and line[13:16].strip() == "CA":
        chain_id = line[21]                       # 链标识 (A, B, ...)
        residue  = AA3TO1[line[17:20].strip()]    # 氨基酸类型
```

**(c) 实验坐标 (Cα 原子的 xyz)**

```python
x = float(line[30:38])   # PDB 固定列格式, 列 31-38
y = float(line[38:46])   # 列 39-46
z = float(line[46:54])   # 列 47-54
```

PDB 格式使用**固定列宽**——每个字段占据固定位置，这源于 1970 年代打孔卡的传统。

#### 3.3 筛选标准与原因

| 参数 | 值 | 原因 |
|------|-----|------|
| 分辨率 | ≤ 2.5Å | 排除低质量晶体（噪音大于预测误差） |
| 序列长度 | 50~250aa | 太短无折叠信息；太长超出 ESM3 显存 |
| 模型 dtype | **float32** | VQ-VAE decoder 权重为 float32, 不支持 BFloat16 混合精度 |

### 5. 结果分析

#### 5.1 ESM3 1.5B 的结构预测精度远低于 AlphaFold2

| 方法 | 输入信息 | 典型 RMSD |
|------|---------|----------|
| **AlphaFold2** | 序列 + MSA + 同源模板 | < 2Å |
| **ESMFold** (ESM-2 3B) | 单序列 | ~5-10Å |
| **ESM3 1.5B** (本实验) | 单序列 | **~8-22Å** |
| 随机卷曲 | — | ~15-30Å |

**原因分析**：

1. **VQ-VAE 离散化瓶颈**：结构被压缩为每残基 1 个整数 token（4096 选 1），连续坐标信息损失巨大。这就像用 4096 种颜色来给照片上色——大致轮廓能辨认，细节全无。

2. **缺少 MSA 输入**：AlphaFold2 的核心优势在于**多序列比对**（MSA，Multiple Sequence Alignment）——它从进化上相关的蛋白序列中提取"这个位置保守性如何""这些残基是否协同变异"等信息，这些是结构预测的最强信号。ESM3 的单序列模式没有这些信息。

3. **模型规模分配**：ESM3 的 1.5B 参数被分配给 5 个 track（序列/结构/SS8/SASA/功能），结构预测只占约 20% 的模型容量。

4. **训练目标**：ESM3 是多任务模型，结构预测是**其中一个子任务**，不是主要优化目标。

#### 5.2 "高置信度假象"：pTM/pLDDT 高 ≠ RMSD 低

```
pdb10ha_A: pTM = 0.89, pLDDT = 0.91  →  RMSD = 22.6Å ← 明显矛盾
pdb10af_A: pTM = 0.89, pLDDT = 0.92  →  RMSD = 18.2Å
```

- pTM 0.89 在 AlphaFold2 中意味着高精度结构（典型 < 2Å）
- 但 ESM3 给出了 pTM=0.89 的同时 RMSD 高达 22Å
- **结论**：ESM3 的 pTM/pLDDT 在单序列模式下不能作为精度指标

**为什么？** pLDDT 和 pTM 是从 VQ-VAE decoder 内部训练出的"自我评分"，模型被训练为对自己的预测自信。问题是——它学会了"自信"但没学会"准确"。这在机器学习中叫**置信度校准失败**（miscalibration）。

#### 5.3 pdb10jy_A 表现最好的可能原因

- 148aa 接近 ESM3 训练数据的平均蛋白长度
- 该蛋白可能属于训练集中覆盖较好的折叠家族
- 1.65Å 分辨率本身有 ~0.2-0.3Å 的实验误差

### 6. 技术要点

#### 6.1 dtype 不匹配 Bug

```python
# 错误做法：模型 BFloat16 + decoder Float32 → 报错
model = model.to(device)          # BFloat16
sd = model.get_structure_decoder().to(device)  # Float32
# RuntimeError: mat1 and mat2 must have the same dtype

# 正确做法：统一使用 Float32
model = model.float().to(device)  # Float32 ✓
sd = model.get_structure_decoder().to(device)    # Float32 ✓
```

原因：`StructureTokenDecoder` 的预训练权重保存为 Float32，且内部有 `torch.mm` 操作要求两个矩阵同 dtype。

#### 6.2 BOS/EOS Token 必须显式设置

```python
struct_ids = out.structure_logits.argmax(dim=-1)
struct_ids[0, 0]  = 4098   # BOS (Beginning Of Sequence)
struct_ids[0, -1] = 4097   # EOS (End Of Sequence)
dec = struct_decoder.decode(struct_ids)   # 否则报错
```

argmax 不一定将 BOS/EOS 位置预测为正确的特殊 token，需手动修正。

#### 6.3 取出 Cα 坐标的正确索引

```python
# bb_pred 形状: (1, L+2, 3, 3)
#               batch, seq, N/CA/C, xyz
ca_coords = dec["bb_pred"][0, 1:-1, 1, :]   # (L, 3)
#                          ↑  ↑    ↑  ↑
#                      batch 0  去首尾 CA  所有xyz
```

## Cell 9: GFP 定点突变效应扫描

### 1. 实验目的

在 GFP 蛋白的 6 个功能关键位点引入**单点突变**（single point mutation，即只改变一个氨基酸），使用 ESM3 推理每个突变体，量化 WT（野生型，Wild Type）与突变体之间的 **embedding 差异**和**结构置信度变化**，评估 ESM3 是否具备**突变效应感知**能力。

### 2. 核心概念 —— 逐词解释

#### 2.1 点突变 (Point Mutation)

蛋白质序列中的一个氨基酸被替换为另一个：

```
WT:    ...MSKGEELFTG...
G67A:  ...MSKGEELFT A ...   ← 仅 G→A 不同
Y66A:  ...MSKGEEL A TG...   ← 仅 Y→A 不同
```

点突变是蛋白质工程中最基本的操作——自然界中，进化正是通过大量随机点突变 + 自然选择实现的。

#### 2.2 GFP 发色团 (Chromophore)

GFP（绿色荧光蛋白）的荧光来源于其内部的**发色团**——由残基 **S65-Y66-G67** 自催化环化形成的对羟基苯亚甲基咪唑酮：

```
Ser65-OH + Tyr66 + Gly67-NH
    │         │
    └──脱水───┘── 氧化 →  发色团 (吸收蓝光, 发出绿光)
```

这个反应**不需要任何辅因子或酶**——只要蛋白正确折叠，发色团就能自发形成。这也是 GFP 成为生物学"万能标记"的原因。

**各突变对发色团的影响**：

| 突变 | 生化效应 | 对荧光的影响 |
|------|---------|------------|
| **S65T** | S→T 增加甲基，加速环化 | **增强**——EGFP 的基础突变，荧光亮 35 倍 |
| **Y66F** | Y→F 仅缺一个 -OH 基团 | **轻微**——发色团仍能形成，光谱几乎不变 |
| **Y66W** | Y→W 侧链更大，共轭体系扩展 | **中等**——已知的 CFP 变体（青色荧光蛋白） |
| **G67A** | G→A 引入甲基，空间位阻 | **破坏**——发色团环化被阻碍，荧光大幅降低 |
| **Y66A** | Y→A：芳香环完全消失 | **破坏**——发色团无法形成，彻底失活 |
| **E222Q** | E→Q：破坏 H⁺ 传递网络 | **中等**——pH 敏感性和亮度改变 |

#### 2.3 Embedding (嵌入向量)

ESM3 对输入的每个氨基酸输出一个 1536 维的向量。这个向量是该残基在整条蛋白上下文中的**语义表示**——在理想情况下，相似的氨基酸（如都是疏水的 L 和 I）在 embedding 空间中应该**距离近**，不同的氨基酸（如疏水的 L 和带正电的 K）应该**距离远**。

**单点突变改变了一个氨基酸 → 突变位点及其周围的 embedding 可能随之偏移。** 衡量这种偏移的指标是本实验的核心。

#### 2.4 五个评估指标 —— 详解

**(a) global_cos (全局余弦相似度)**

```python
global_cos = cosine_similarity(
    mut_emb.mean(dim=0),    # 突变体所有 238 个位置的 1536 维向量 → 取平均 → 1 个 1536 维
    wt_emb.mean(dim=0)      # WT 同理
)
```

**含义**：整条蛋白的"平均表示"在突变前后有多相似？cos = 1 表示完全相同，cos < 1 表示有偏移。

**局限**：均值池化将 238 个位置的差异压缩为 1 个数字。1 个氨基酸的变化被 237 个未变氨基酸**稀释**到几乎不可检测。因此 global_cos 极其不敏感——所有突变都接近 1.0000。

**(b) local_cos (局域余弦相似度)**

```python
ls = max(0, pos - 5)        # 突变位点前 5 个残基
le = min(L, pos + 6)        # 突变位点后 5 个残基 (共约 11 个)
local_cos = cosine_similarity(
    wt_emb[ls:le].flatten(),  # 11×1536 = 16896 维向量
    mut_emb[ls:le].flatten()
)
```

**含义**：只看突变位点**附近**（±5 残基）的 embedding 变化。这是本实验中最敏感的指标，能捕捉到全局均值池化无法检测的微弱局部信号。

**与 global_cos 的对比**：
- global_cos 看整条蛋白 → 对 1/238 的变化不敏感
- local_cos 看 ±5 残基 → 对 1/11 的变化敏感 ~20 倍

**(c) pTM Δ (全局结构置信度变化)**

```python
pTM_delta = mut_ptm - wt_ptm
```

**含义**：突变后，模型对自己预测的全局结构置信度（pTM）的变化。如果突变破坏了蛋白折叠，pTM 应下降（Δ < 0）。正 Δ 表示模型"认为"突变后的结构更可信。

**(d) pLDDT Δ (位点) (突变位点置信度变化)**

```python
plddt_delta = mut_plddt[pos] - wt_plddt[pos]
```

**含义**：突变位置本身的结构置信度变化。只比较这一个位置。

**(e) 局域 pLDDT Δ (±5残基)**

```python
plddt_local_delta = mut_plddt[ls:le].mean() - wt_plddt[ls:le].mean()
```

**含义**：突变位点 ±5 残基范围内平均 pLDDT 的变化。比单点 pLDDT 更稳定（不容易被一个异常值带偏），能反映突变对其**周围区域**结构置信度的影响。

#### 2.5 余弦相似度 (Cosine Similarity)

$$	ext{cosine\_similarity}(\mathbf{a}, \mathbf{b}) = rac{\mathbf{a} \cdot \mathbf{b}}{\|\mathbf{a}\| \|\mathbf{b}\|}$$

**直觉理解**：余弦相似度衡量两个向量"指向"同一方向的程度：
- cos = 1：完全同向（向量之间夹角 0°）
- cos = 0：正交（夹角 90°）
- cos = -1：完全反向（夹角 180°）

在 embedding 空间中，cos 高说明突变没有改变模型的"理解"；cos 低说明突变显著改变了模型的内部表示。

**与欧几里得距离的区别**：余弦相似度对向量的**长度**（magnitude）不敏感——它只看方向。这在 embedding 比较中是优点，因为 embedding 的绝对值大小往往受位置、氨基酸类型等噪声影响，而**方向**更能反映语义变化。

### 3. 实验设计

#### 3.1 推理管线

```
GFP WT 序列 (238aa)
    │
    ├─→ ESM3 → wt_emb (238, 1536)  +  wt_pLDDT + wt_pTM
    │
    ├─ Y66A 突变体 → ESM3 → mut_emb → 计算各 Δ 指标
    ├─ Y66F 突变体 → ESM3 → ...
    ├─ Y66W 突变体 → ESM3 → ...
    ├─ S65T 突变体 → ESM3 → ...
    ├─ G67A 突变体 → ESM3 → ...
    └─ E222Q 突变体 → ESM3 → ...
```

每个突变体独立加载 ESM3 进行一次推理（共 1 WT + 6 mut = 7 次推理）。

#### 3.2 突变创建

```python
def make_mutant(wt_seq, pos, mut_aa):
    # pos 是 0-indexed (Python string convention)
    return wt_seq[:pos] + mut_aa + wt_seq[pos+1:]

# 示例
make_mutant("MSKGE", 1, "T")  # → "MTKGE"   (S→T at 0-indexed position 1)
```

#### 3.3 关键：0-indexed vs 1-indexed 位置编号

```python
# Python string:  0-indexed
gfp[64]  # → "S"  (这是 PDB 编号中的 Ser65)

# PDB/生物学惯例: 1-indexed
# 输出时显示 pos+1 以匹配生物学惯例
print(f"Position {pos+1}: {wt}->{mut}")
```

### 5. 结果分析

#### 5.1 方向正确但信号微弱

```
破坏性突变 avg local_cos: 0.9961
保守性突变 avg local_cos: 0.9982
差值:                      0.0021
```

破坏性突变的 local embedding 偏移比保守性突变大 0.0021。这个差异极其微小（千分之二），但**方向正确**。

**为什么 ESM3 1.5B 对突变不敏感？**

1. **参数量不足**：1.5B 参数需支撑 5 个轨道，序列 track 的容量不足以精细编码单点突变的影响
2. **无 MSA**：缺少这个位置 "进化保守性" 的先验——如果是高度保守位点（如发色团核心 Y66），突变应引起巨大变化
3. **训练数据无配对 WT/mutant 对比**：ESM3 从未被训练为 "比较 WT 和突变体"——它只见过单独的序列
4. **1536 维 too large for 1 mutation**：一个氨基酸的变化散布在 1536 维空间中，大部分维度不受影响

#### 5.2 E222Q：最意外的强信号

```
E222Q: local_cos = 0.9885 (最低!)  pLDDT_local_Δ = -0.122
```

E222Q 距离发色团（pos 64-66）约 156 个残基，在空间上也不相邻。但它产生了最强的 embedding 和 pLDDT 变化。

**可能原因**：
- E222Q 位于蛋白表面（溶剂暴露区域），pLDDT 在表面区域天然波动较大
- Glu222 是带负电的谷氨酸，Gln222 是中性的谷氨酰胺——电荷变化引入的静电扰动比疏水替换（Y→A）更显著
- ESM3 的 embedding 可能对**电荷变化**比**大小变化**更敏感

#### 5.3 global_cos 完全无效

所有突变的 global_cos >= 0.9998，几乎无变化。均值池化将 238 个位置的 1536 维信息压缩为 1 个 1536 维向量，1 个氨基酸的差异被 237 个未变氨基酸完全稀释。

**结论**：单点突变效应评估必须使用**局部窗口**，全局池化不是正确的工具。

#### 5.4 pTM 波动无规律

```
G67A: pTM -0.042  (破坏性突变, pTM 下降 —— 方向正确!)
Y66A: pTM -0.007  (同为破坏性, 几乎不变)
Y66F: pTM +0.005  (保守替换, 反而略增)
```

pTM 变化与突变严重性无单调关系。对于功能评估，pTM 在 1.5B 版本上不可靠——波动可能由随机噪声主导。

#### 5.5 E222Q 结构置信度下降最多的可能解释

```
pdb10af_A:  E222Q pLDDT local Δ = -0.122
pdb10jy_A:  G67A pLDDT local Δ = +0.001
```

E222 暴露在蛋白表面，pLDDT 倾向在表面区域高估置信度。突变可能改变表面水合模式，影响局部结构稳定性——模型捕捉到了这个信号。

### 6. 技术要点

#### 6.1 0-indexed vs 1-indexed

```python
# Python string: 0-indexed
gfp[64]  # "S" → Ser65 (生物学编号)

# 突变定义使用 0-indexed
("S65T", 64, "S", "T", ...)

# 输出显示使用 1-indexed (生物学惯例)
print(f"Pos {pos+1}: {wt}->{mut}")
```

#### 6.2 局域 embedding 窗口选择

±5 残基是一个经验选择：
- 太小：捕捉不到突变对邻近结构的影响范围
- 太大：退化为类似 global_cos 的稀释效应
- ±5 约覆盖突变位点起 1-2 圈 α-helix 的范围（3.6 残基/圈）

#### 6.3 float32 同样需要

与 Cell 8 相同——VQ-VAE structure decoder 要求 float32。

## 综合讨论

### ESM3 1.5B 的能力边界总结

| 能力 | 表现 | 是否可用于论文/研究 |
|------|------|-------------------|
| 掩码填空 (MLM) | 良好 (17% argmax, 46% top-5) | 可用于蛋白质序列工程 |
| 二级结构预测 (SS8) | 良好 (GFP β-barrel 精确) | 可用——Cell 2 已验证 |
| SASA 预测 | 可用 | 可用于可溶性分析 |
| 3D 结构预测 (RMSD) | 很差 (8-22Å) | **不可用于精度要求** |
| 单点突变感知 | 极微弱 (cos > 0.99) | 需要更大模型 (98B) |
| pTM/pLDDT 置信度 | 不可靠 | **不可用于质量评估** |
| PDB 生成 (可视化) | 可用 | 教学/演示用途 |

### 正确使用 ESM3 的心态

- ESM3 是一个**生成式蛋白设计模型**，设计用于多模态蛋白质生成，不是高精度结构预测工具
- 1.5B 版本是开源的最弱版本——与 98B 完全版有数量级性能差距
- 对需要定量精度的任务，建议使用专门工具：
  - 结构预测 → AlphaFold2/3 或 ESMFold（ESM-2 + folding trunk）
  - 突变效应 → ESM-1v 或 Tranception（专门训练的变异效应模型）
  - 蛋白设计 → ProteinMPNN + RFdiffusion
