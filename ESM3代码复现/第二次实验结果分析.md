# 第二次实验结果分析



## 1. ESM3 不是纯 MLM 模型

这是最关键的一点。ESM3 的训练目标是多轨道的：

```
ESM3 训练 loss = sequence_loss + structure_loss + ss8_loss + sasa_loss + function_loss
                         ↑                          ↑              ↑           ↑
                    只占 ~20%               结构预测占大头          你激活的那些 track

ESM-2 训练 loss = sequence_loss  ← 100% 精力在填氨基酸上
```

ESM3 的 1.5B 参数被分配给了 5 个 track，序列 track 只是其中一部分。相比 ESM-2（同参数量的纯序列模型，掩码准确率可达 30-40%），ESM3 的序列预测天生就更弱——因为它把容量给了结构预测。你 Cell 2 的结构预测质量很好（pLDDT=0.64，β-barrel 完美），代价就是序列 track 精度下降。

------

## 2. 你的评估基准"随机 5%"是错的

氨基酸有 20 种，但 tokenizer 的 vocab 有 33 个 token（20 种氨基酸 + 13 个特殊 token）。随机猜的准确率：

| 基线                     | 准确率    |
| ------------------------ | --------- |
| 20 种氨基酸均匀随机      | 5.0%      |
| 33 种 token 均匀随机     | 3.0%      |
| 按自然频率猜（总是猜 L） | 9.6%      |
| **ESM3 掩码预测**        | **17.1%** |
| **ESM3 Top-5 命中**      | **45.7%** |

17.1% 是随机的 3-6 倍，Top-5 有近一半的 mask 位置在 5 个候选内找到正确答案。

------

## 3. 正确答案"平均排名第 7"

这意味着模型把正确答案排到了第 7 位，而不是第 1 位。这是蛋白质的特性——同一个 context 下多个氨基酸在物理化学上等价：

```
位置 7 (L):  模型预测 K (碱性) 而不是 L (疏水)
             → 但真正该在位置 7 的是"大侧链"类氨基酸
             → L 排名 9 不是因为模型瞎猜，而是 I/V/A/F 都符合这个上下文
```
